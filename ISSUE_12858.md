# Issue #12858: [Bug] User skills do not load in v1.3.0

**Status:** OPEN
**Author:** Jared Willard (@jdwillard19)
**Created:** 2026-02-13
**Labels:** bug
**Difficulty:** Medium | **Impact:** Functionality broken
**Recommendation:** ⭐ Good issue - reproducible on macOS with dev mode setup

---

## Bug Description

When I try to add skills to `~/.openhands/skills/`, they do not show up in the OpenHands web application conversation.

## Expected Behavior

Skills should show up when I add them to `~/.openhands/skills/`

## Actual Behavior

Added skills do not show up

---

## Steps to Reproduce

1. Download v1.3.0 OpenHands code
2. Add skills to `~/.openhands/skills`
3. Run `make build` and `make run`
4. Open web application in Firefox (also tested Chrome), then configure LLM settings (Sonnet 4.5)
5. Start a conversation from scratch
6. Show available skills does not show added skills

---

## Environment Information

- **Installation Method:** Development workflow
- **OpenHands Version:** 1.3.0
- **Model:** `us.anthropic.claude-sonnet-4-5-20250929-v1:0`
- **Operating System:** macOS
- **Browser:** Firefox 140.7.0esr (also tested Chrome)
- **Running:** Developer mode

## Analysis

The critical log line shows:
```
Loaded 42 skills from agent-server: sources={'sandbox': 1, 'public': 41, 'user': 0, 'org': 0, 'project': 0}
```

The `'user': 0` indicates that the skill loader is not finding any user skills, even though the logs show symlinks being created to user skill files:
- `deepsolve.md`
- `eda.md`
- `scope_propose.md`
- `hypothesis_propose.md`
- `workplan_propose.md`
- `checkpoint.md`

This suggests a problem in the skill loading logic at `skill_loader.py:356`.
